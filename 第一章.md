#### 第一章：什么是JavaScript

1. 简短的历史回顾
   + JavaScript首先被应用为客户端表单验证
2. JavaScript实现
   + 完整的JavaScript实现包含以下几个部分
     + 核心（ECMAScript）
     + 文档对象模型（DOM）
     + 浏览器对象模型（BOM）
   + ECMAScript
     + 宿主环境
       1. Web浏览器
       2. 服务器端平台Node.js
       3. Adobe Flash (基本被淘汰)
     + ECMA-262定义了什么
       	1. 语法
       	1. 类型
       	1. 语句
       	1. 关键字
       	1. 保留字
       	1. 操作符
       	1. 全局对象
     + ECMAScript版本
       1. es4被废除
       2. es6于2015年6月发布，正式支持了类、模块、迭代器、生成器、箭头函数和众多新的数据类型
       3. 现在最新为es12（2022）
     + ECMAScript符合性
       1. 支持ECMA-262的所有语法
       2. 支持Unicode字符标准
       3. 增加额外的内容（类型、值、对象、属性和函数）
       4. 支持未定义的程序和正则表达式语法
     + 浏览器对ECMAScript的支持
       1. 大多数浏览器都兼容
   + DOM
     + 文档对象模型是一个应用编程接口，用于在HTML中使用扩展的XML。
     + DOM通过创建表示文档的树，让开发者可以随心所欲的控制网页的内容和结构（如轻松的删除、添加、替换、修改节点）
     + 万维网联盟制订了DOM标准的进程
   + DOM级别
     + DOM Level1 : 包含 DOM Core 和 DOM HTML，目标是映射文档
     + DOM Level2新增：
       + DOM视图：描述追踪文档不同视图的接口。
       + DOM时间：描述时间及时间处理的接口。
       + DOM样式：描述处理元素CSS样式的接口。
       + DOM遍历和范围：描述遍历和操作DOM树的接口。
     + DOM Level3进一步扩展了DOM：
       + 增加了以统一的方式加载和保存文档的方法，验证文档的方法
       + DOM Core经过扩展支持了所有XML1.0的特性。
   + 其它DOM
     + 可伸缩矢量图（SVG）
     + 数学标记语言（MathML）
     + 同步多媒体集成语言（SMIL）
   + 支持情况
     + 大多数浏览器都兼容
   + BOM
     + 浏览器对象模型，用于支持访问和操作浏览器的窗口。
     + 使用BOM，开发者可以操控浏览器显示页面之外的部分。
   + BOM针对对象
     + 浏览器窗口和子窗口
     + 弹出新浏览器窗口的能力
     + 移动、缩放和关闭浏览器窗口的能力
     + navigator对象，提供关于浏览器的详尽信息
     + location对象，提供浏览器加载页面的详尽信息
     + screen对象，提供关于用户屏幕分辨率的详尽信息
     + performance对象，提供浏览器内存占用，导航行为和时间统计的详尽信息
     + 对cookie的支持
     + 其它自定义对象，如XMLHttpRequest
   + JavaScript版本
     + Mozilla Foundation在持续开发 JavaScript，为它增加新特性、关键 字和语法，所以 JavaScript 的版本号也在不断递增
     + JavaScript2.0作为一个目标已经不存在了
   + 小结
     + JavaScript是一门用来与网页交互的脚本语言，包含以下三个组成部分
       1. ECMAScript：由ECMA-262定义并提供核心功能
       2. 文档对象模型（DOM）：提供与网页内容交互的方法和接口
       3. 浏览器对象模型（BOM）：提供与浏览器交互的方法和接口
